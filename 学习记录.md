## Pydantic

1. 数据校验（validation）：检查字段类型、必填/可选、范围、正则、枚举等；错误信息可读、可定位到字段路径。
2. 数据解析/类型转换（parsing/coercion）：把 "123" 转成 int、把字符串日期转成 datetime、把 dict 转成嵌套对象等（也可配置为严格模式不允许隐式转换）。
3. 结构化建模（models）：定义嵌套结构、默认值、别名（alias）、只读/私有字段、计算字段等。
4. 序列化/反序列化：model ↔ dict/JSON；支持字段过滤、排除 None、别名输出等。
5. 配置与环境变量（Settings）：把环境变量、.env、配置文件映射成强类型配置对象（常用于服务配置）。
6. 生成 JSON Schema：从模型直接导出 schema；在 Web 框架里用于文档与校验。
7. 和 FastAPI 等框架集成：请求体/响应体校验、自动文档（OpenAPI）基本都依赖 Pydantic 的模型与 schema。
8. 自定义校验逻辑：字段级/模型级校验器（validators），跨字段约束（例如 end_time 必须晚于 start_time）。
9. 更好的类型边界：让“输入边界”（API、消息队列、DB 读写层、配置）有清晰契约，减少运行时类型错误。
